# ARCHITECTURE.md

## Purpose

Document the system boundary, major components, data flow, external services, and architecture-change rules.

## Required Content

Each project must describe:
- user/client interfaces
- application and service layers
- persistent data and migrations
- external services and trust boundaries
- authentication and authorization boundaries
- deployment topology

## Change Rule

Major architecture changes require the reason, affected files, risks, migration steps, validation plan, and rollback plan.
